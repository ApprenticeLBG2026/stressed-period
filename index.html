💼 Stressed Period Identification for IRC

Automated Credit Risk Calibration for Internal Model Method

Overview

This project automates the identification of credit-stressed periods to support Internal Model Method (IMM) calibration for IRC (Incremental Risk Charge).

Under CRR Article 284(4), banks using internal models must calibrate stressed inputs using an appropriate historical stress period. This project replaces a manual, spreadsheet-based process with a reproducible Python pipeline.

🔒 Regulatory Framework: CRR Article 284(4) | Internal Model Method (IMM) | Incremental Risk Charge (IRC)

Business Context

Banks using internal models for capital calculations must:

  • Identify and calibrate stressed periods from historical data
  • Use appropriate PD (Probability of Default) time series
  • Incorporate exposure-weighted EAD (Exposure at Default) data
  • Ensure reproducibility and auditability of calculations
  • Comply with regulatory validation requirements

What the Pipeline Does

Input Processing

  • Ingests daily PD time series by ticker
  • Loads exposure-weighted EAD data
  • Validates data quality and completeness

Stressed Period Computation

  • Computes alternative stressed-period metrics
  • Identifies historical stress windows
  • Calculates shock magnitudes and durations

Output Generation

  • Produces reproducible visualization plots
  • Generates summary statistics tables
  • Exports results for model validation

Key Outputs

The pipeline generates the following deliverables:

  • assets/stressed_period_plot.png – Historical PD trends with identified stress periods
  • Stress metrics summary table – Period statistics and shock calculations
  • Reproducible notebooks – Full audit trail of methodology

Core Pipeline Code

Example of the stressed period identification logic:

Python - Stress Period Identification
""" Core pipeline for identifying stressed periods using PD time series """ import pandas as pd import numpy as np def compute_pd_shocks(pd_series, window=252): """ Compute rolling PD shocks (changes) over time. Args: pd_series: Daily PD time series window: Rolling window in business days (default: 252 = 1 year) Returns: DataFrame with rolling statistics """ rolling_mean = pd_series.rolling(window).mean() rolling_std = pd_series.rolling(window).std() shocks = (pd_series - rolling_mean) / rolling_std return shocks def identify_stress_periods(pd_series, threshold=2.0): """ Identify periods where PD exceeds stress threshold. Args: pd_series: Daily PD time series threshold: Std dev threshold for stress identification Returns: Boolean series marking stressed periods """ shocks = compute_pd_shocks(pd_series) stressed = np.abs(shocks) > threshold return stressed # Usage example pd_data = pd.read_csv('daily_pd_by_ticker.csv', index_col='date') stress_flags = identify_stress_periods(pd_data['PD'])
